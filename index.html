Documentation & Notes

I. Purpose

ResearchOS was built out of necessity for the Postgraduate Research Hub Discord community. It is designed for postgrads drowning in the trenches of paper deadlines, providing an unopinionated, dense interface to execute queries, process literature, and manage knowledge.

II. Architecture & Security

This is a two-layer application. The frontend operates entirely statically via HTML/CSS/JS. The backend relies on Vercel Serverless Functions to securely route Exa API requests.

Auth & Data: Managed by Supabase. Your private data is secured via Row Level Security (RLS). No service keys are exposed to the client. Environment variables (SUPABASE_URL, SUPABASE_ANON_KEY, EXA_API_KEY) are injected dynamically at runtime.

III. Exa Protocol

The core pipeline queries the Exa neural search engine.

  • Semantic Search: Leverages Exa's neural search type. Phrase queries organically (e.g., "Papers detailing the mechanisms of neuroplasticity during REM sleep").
  • Limitations: Search depth is locked to reputable academic domains (arXiv, PubMed, bioRxiv, Nature, ScienceDirect, Scholar, PLOS, Cell). Summaries are AI-generated; always cross-reference the original manuscript.
